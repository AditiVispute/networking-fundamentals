# OSI Model (Open Systems Interconnection)

## 📌 Overview

The OSI Model is a conceptual framework used to understand how data travels across a network from one device to another.

It consists of 7 layers, each responsible for a specific function in the communication process.

---

## 🧩 7 Layers of the OSI Model

### 1️⃣ Physical Layer
- Transmits raw bits over cables or wireless signals.
- Deals with hardware elements like cables and switches.

### 2️⃣ Data Link Layer
- Responsible for node-to-node communication.
- Handles MAC addressing and error detection.

### 3️⃣ Network Layer
- Responsible for routing data packets.
- Uses IP addresses to determine the best path.

### 4️⃣ Transport Layer
- Ensures reliable or fast data transmission.
- Protocols: TCP (reliable) and UDP (fast).

### 5️⃣ Session Layer
- Manages sessions between applications.

### 6️⃣ Presentation Layer
- Handles encryption, compression, and formatting.

### 7️⃣ Application Layer
- Closest to the user.
- Includes protocols like HTTP, FTP, SMTP.

---

## 🌍 Real World Example

When accessing a website:

- Application layer sends HTTP request.
- Transport layer ensures delivery using TCP.
- Network layer routes the packet using IP.
- Physical layer transmits signals through cables.

---

## 🎯 Key Takeaways

- OSI model helps understand network communication.
- Each layer has a specific responsibility.
- Useful for troubleshooting and cybersecurity.
